## 走进调试器开发

从本章开始，我们正式走进调试器开发的世界。

本章内容分为四个部分：

- **需求分析**：从功能性需求和非功能性需求两个维度，分析现代调试器应该具备的核心能力
- **架构设计**：介绍调试器的分层架构设计，以及如何实现良好的扩展性和命令管理
- **实现方案**：确定具体的实现策略，包括指令级和符号级两个版本的调试器实现
- **功能演示**：通过实际运行效果，展示调试器的各项功能和交互体验

本章我们将从需求分析切入，探讨开发一个调试器时需关注的核心要点，随后逐步梳理调试器的技术架构，甄选合适的实现方案，最终实现一个能够运行的调试器雏形。

我们将在此基础上继续深入，逐步实现指令级调试、符号级调试，带领读者一步步理解调试器的工作原理和设计实现。

>本章结束后，读者也将掌握使用 [spf13/cobra](https://github.com/spf13/cobra) 进行命令行程序开发的一些技巧，如命令管理、选项管理、参数管理、help信息管理、如何实现自动补全等等。go语言经常用来开发一些使用的命令行工具，而spf13/cobra 就是当前最成功的一个项目，很多知名项目的命令行工具是基于它构建，比如 kubernetes、hugo、Github CLI 等等。
